# Failure Recovery

## 它解决什么

失败恢复让机器人在动作出错后发现问题、定位原因，并重新回到可执行状态。真实机器人系统里，失败恢复往往比单次成功更重要。

## 常见失败

- 抓空。
- 抓住但滑落。
- 物体被遮挡。
- 轨迹碰撞。
- 目标位置不可达。
- 语言目标不明确。
- 传感器异常。
- 人或环境突然变化。

## 恢复流程

```text
执行动作
→ 检测异常
→ 判断失败类型
→ 更新状态
→ 选择恢复策略
→ 重试或请求人工接管
```

## 恢复策略

- 重新感知。
- 换视角。
- 调整抓取点。
- 降低速度。
- 重新规划路径。
- 切换备用技能。
- 请求人类确认或接管。

## 为什么难

- 很多失败不是立即可见。
- 失败原因可能有多个。
- 重试可能带来安全风险。
- 数据集中常常缺少失败和恢复过程。

## 初学者记忆点

不要只问“模型能不能做成”。还要问“做不成时，它知不知道自己做不成，以及下一步怎么办”。
