# Data Lifecycle

## 它解决什么

机器人数据不是采完就能训练。它需要经历采集、清洗、标注、切分、训练、评测和回流。任何一个环节出问题，都会影响策略质量。

## 生命周期

```text
任务设计
→ 数据采集
→ 数据清洗
→ 标注和对齐
→ 数据格式化
→ 训练集/验证集/测试集切分
→ 模型训练
→ 实机评测
→ 失败数据回流
```

## 关键字段

一个 episode 通常应该记录：

- 任务指令。
- 时间戳。
- 图像、深度、触觉或其他传感器。
- 机器人本体状态。
- 动作命令。
- 成功或失败标签。
- 人工干预。
- 环境和物体信息。

## 数据质量问题

- 图像和动作不同步。
- 任务成功标签不准确。
- 只记录成功样本，缺少失败恢复。
- 不同机器人动作空间不统一。
- 数据集训练集和测试集场景过于相似。
- 人类示范不适合机器人本体直接模仿。

## 数据飞轮

理想的数据飞轮是：

1. 机器人执行任务。
2. 系统记录成功、失败和人工接管。
3. 人类或自动工具标注关键失败。
4. 新数据进入训练。
5. 新策略上线评测。
6. 继续收集长尾问题。
