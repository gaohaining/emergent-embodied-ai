# Perception Pipeline

## 它解决什么

感知流水线把原始传感器信号变成任务可用的信息。机器人真正需要的通常不是“这张图里有什么”，而是“我下一步应该如何行动”。

## 典型流水线

```text
传感器输入
→ 标定和同步
→ 低层特征
→ 对象、空间、接触状态
→ 任务相关状态
→ 策略或规划器使用
```

## 每一步在做什么

| 环节 | 作用 | 常见问题 |
| --- | --- | --- |
| 标定 | 知道相机、机械臂、世界坐标之间的关系 | 标定误差会导致抓取偏差 |
| 同步 | 让图像、深度、触觉、本体状态对齐 | 延迟会让控制用到过期状态 |
| 检测与分割 | 找到物体和区域 | 遮挡、反光、相似物体 |
| 姿态估计 | 估计物体 3D 位姿 | 对称物体、透明物体难 |
| 状态估计 | 判断任务进度和接触状态 | 失败状态不易观测 |
| 可供性估计 | 判断哪里可以抓、推、放 | 需要和动作数据对齐 |

## 感知和动作的关系

在具身智能里，感知不是独立模块。它要回答动作相关问题：

- 这个物体在哪里？
- 它能不能被当前夹爪抓住？
- 抓取后是否稳定？
- 目标区域是否可达？
- 当前动作是否失败？

## 失败模式

- 识别对了，但 3D 位置错了。
- 物体被遮挡，系统误以为不存在。
- 透明或反光物体导致深度错误。
- 相机延迟导致动作落后于真实状态。
- 感知模型在训练场景表现好，换光照或背景后失败。
