# 09 Frontier Tracking

这个目录用于持续跟踪前沿动态。

## 文件

- [inbox.md](inbox.md)：临时收集所有新材料。
- [papers.md](papers.md)：论文跟踪表。
- [model-paper-notes.md](model-paper-notes.md)：模型谱系代表论文笔记（LLM/VLM/VAM/VLA/World Model/WAM/动作生成头）。
- [tooling-and-system-stack.md](tooling-and-system-stack.md)：具身工程常用工具链与系统栈（感知、标定、规划、仿真）。
- [community-insights.md](community-insights.md)：技术社区精彩文章（知乎、Medium、机器之心等）。
- [labs-and-companies.md](labs-and-companies.md)：实验室、公司和产品动态。
- [datasets-and-benchmarks.md](datasets-and-benchmarks.md)：数据集、benchmark 和评测动态。
- [open-source.md](open-source.md)：开源项目和工具链。
- [repo-watchlist.md](repo-watchlist.md)：一手来源工作流（论文、项目页、代码仓库、数据集、benchmark）。
- [source-map.md](source-map.md)：外部材料的信息源类型和收集规则。
- [templates/weekly-review.md](templates/weekly-review.md)：周度回顾模板。
- [templates/repo-card.md](templates/repo-card.md)：代码仓库分析模板。
- [templates/source-item.md](templates/source-item.md)：外部材料记录模板。

## 每周处理规则

1. 清理 inbox。
2. 选出 3-5 个最值得精读的材料。
3. 更新对应主题页。
4. 记录本周判断变化。

## 外部材料使用原则

- 优先使用论文、项目页、官方代码仓库、数据集页面和 benchmark 页面。
- 工程类项目优先记录可运行性、数据格式、硬件接口和复现成本。
- 对高频更新内容，记录最后复查日期，避免过期链接混进稳定结论。
- 社区讨论、中文长文和搜索结果主要作为线索，关键事实尽量回到论文、项目页、官方博客、代码或数据集页面确认。
