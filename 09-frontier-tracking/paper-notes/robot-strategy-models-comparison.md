# 机器人策略模型论文阅读比较

Status: draft

Last checked: 2026-06-03

这份笔记整理机器人策略模型路线的代表论文，不把路线强绑定为 Manipulation。更准确的读法是：这些论文共同讲的是 **机器人策略模型如何从手工几何、离线示范和技能库，走向通用视觉-语言-动作模型、生成式动作头、世界模型和部署中自我提升**。

操作任务之所以频繁出现，是因为它们最能暴露具身智能的硬问题：空间、接触、遮挡、长程依赖、数据成本、安全和实时控制。但同一批思想也影响移动操作、腿式导航、人形机器人、服务机器人和云端机器人平台。

逐篇精读版见 [robot-strategy-models-detailed.md](robot-strategy-models-detailed.md)。本页保留为横向比较和阅读路线。

## 一句话路线

早期系统先解决“看见哪里可以抓、哪里可以放”。随后，研究重点转到“如何从示范中学动作”“如何收更多、更好的数据”“如何把语言和视觉知识接到动作上”。VLA 出现后，问题又变成：动作应该是 token、连续轨迹、diffusion、flow，还是由世界模型先想象后果再行动？2025-2026 的新变化是，VLA 不再只靠模仿学习，开始用 RL、人在回路、世界模型生成数据和开源工具链形成数据飞轮。

## 怎么读这批论文

先不要问“谁取代谁”，而要问每篇论文主要打通哪个关键转换：

| 关键转换 | 代表论文 | 它改变的问题 |
| --- | --- | --- |
| 感知和可供性 | GraspNet、CLIPort、SayCan | 机器人如何知道当前世界里什么动作可执行。 |
| 示范数据和工具 | robomimic、ALOHA、Mobile ALOHA、UMI、LeRobot | 人类示范如何变成可复用、可扩展、可训练的数据。 |
| 动作生成头 | ACT、Diffusion Policy、DP3、RDT、FAST、pi0 | 动作是一帧一帧回归，还是一段轨迹、一个分布、一个 token 序列或一个 flow。 |
| 通用策略和 VLA | RT-1、RT-2、Open X、Octo、OpenVLA、HPT、SmolVLA、GR00T | 如何让策略跨任务、跨物体、跨本体，并继承视觉语言模型的知识。 |
| 世界模型和经验学习 | DreamerV3、UniSim、Cosmos、pi*0.6、GigaBrain | 如何用预测、生成数据、离线/在线 RL 打破纯模仿学习上限。 |

## 论文故事卡

### 1. GraspNet-1Billion

- 故事：这篇站在比较传统的抓取问题上，但它的重要性不只是“抓取”。它说明机器人学习需要大规模、标准化、可比较的数据和评测，否则不同方法很难判断谁真的更稳。
- 方法要点：给真实场景和大量对象提供密集抓取姿态标注，把抓取从少量手工例子推进到大规模 benchmark。
- 证据：数据集和 benchmark 证据强，适合评估抓取检测；但它主要覆盖抓取候选，不等于完整任务策略。
- 读完记住：几何抓取没有消失，它仍是很多系统的可靠底座；后续 VLA 再强，也常要落到可执行位姿和接触约束。
- 局限：抓取只是动作链的一环，不覆盖语言目标、长程任务、失败恢复和跨场景部署。

### 2. CLIPort

- 故事：CLIPort 解决的是“语言知道是什么，但机器人还要知道放在哪里、怎么动”。它把 CLIP 的开放词汇语义和 Transporter 的空间操作结构结合起来。
- 方法要点：一个 what pathway 负责语义，一个 where pathway 负责空间匹配，输出 pick/place 类动作。
- 证据：在语言条件的 tabletop 任务中展示较强泛化；证据主要围绕结构化操作任务。
- 读完记住：VLM 接机器人时，不能只保留语义 embedding，还要有空间对齐机制。
- 局限：适合 pick/place 风格任务；对连续力控、复杂接触和长程多阶段任务覆盖有限。

### 3. robomimic

- 故事：它不是一个炫酷模型，而是告诉你离线人类示范学习到底哪些因素重要。对于知识库很关键，因为很多后来论文都依赖“用示范训练策略”这个基本假设。
- 方法要点：统一离线示范数据、算法、评测和分析，比较 BC、RNN、Transformer、视觉输入、状态输入等变量。
- 证据：benchmark 和离线示范实验比较系统；强在方法学，弱在通用智能叙事。
- 读完记住：机器人模仿学习的成败常常不只由模型决定，数据质量、观测、动作空间、评测协议同样决定结果。
- 局限：benchmark 与真实部署仍有距离；不能直接说明跨本体和开放语言泛化。

### 4. SayCan

- 故事：LLM 会说“应该做什么”，但机器人未必“能做”。SayCan 的核心是把语言合理性和物理可执行性结合。
- 方法要点：LLM 给候选技能打语义分，低层技能给 affordance/value 分，两者组合后选择下一步。
- 证据：真实移动操作机器人上的长程语言任务；证据强在任务规划和技能选择，不在端到端低层控制。
- 读完记住：SayCan 不是让 LLM 直接控制电机，而是让 LLM 在技能库上做有物理约束的选择。
- 局限：依赖预定义技能库；技能库之外的动作不会凭语言自然涌现。

### 5. RT-1

- 故事：RT-1 把“大规模真实机器人数据 + Transformer 策略”变成一个清晰实验命题。它说明策略可以从多任务真实数据中学到一定泛化。
- 方法要点：输入图像和任务条件，输出离散动作 token；通过真实机器人数据规模化训练。
- 证据：Google 机器人数据、多任务实验和数据规模/模型规模消融；证据较强但本体和数据分布相对集中。
- 读完记住：RT-1 是 VLA 前的关键台阶，先证明了 robot transformer 在真实数据上能扩展。
- 局限：开放语义和互联网知识迁移还不充分；跨本体能力不是核心。

### 6. ChatGPT for Robotics

- 故事：这篇更像设计原则而不是机器人策略学习论文。它讨论 ChatGPT 如何通过 prompt、API 和安全约束参与机器人任务。
- 方法要点：把机器人能力包装成可调用函数或代码接口，让 LLM 生成计划或控制胶水层。
- 证据：原型和案例展示为主；适合启发系统设计，不适合作为性能基准。
- 读完记住：LLM 可以当“任务编排层”，但必须有明确 API、约束和人类监督。
- 局限：容易被过度解读成“LLM 直接会控制机器人”；实际低层执行仍靠传统控制或学习策略。

### 7. ACT / ALOHA

- 故事：ACT 解决逐帧动作预测的累积误差问题；ALOHA 让低成本双臂硬件和高质量遥操作数据可用。
- 方法要点：Action Chunking Transformer 一次预测一段动作序列，并通过 temporal ensembling 平滑执行。
- 证据：真实双臂精细操作任务，少量示范下成功率高；对低成本硬件实践影响很大。
- 读完记住：动作分块是后续很多机器人策略的基本技巧，它把“下一步动作”改成“短时动作计划”。
- 局限：主要在受控任务和特定硬件上验证；泛化到开放环境仍需要更多数据和系统设计。

### 8. Diffusion Policy

- 故事：很多动作不是单峰答案。比如绕过障碍可以从左也可以从右，传统回归容易预测到中间的坏动作。Diffusion Policy 把动作建模为条件生成分布。
- 方法要点：从噪声迭代去噪生成动作轨迹，结合视觉条件和 receding horizon control。
- 证据：多个 benchmark 和真实任务，报告显著超过当时 SOTA；代码和数据开放，证据较扎实。
- 读完记住：它让“生成一段合理动作轨迹”成为机器人策略学习的主流思路。
- 局限：迭代采样带来延迟；安全、恢复和长程任务还要靠外部系统。

### 9. DreamerV3

- 故事：DreamerV3 不专门为机器人操作而写，但它代表世界模型 RL 的强基线：先学环境动态，再在 latent imagination 中训练策略。
- 方法要点：学习 latent dynamics、reward、value，并用想象轨迹优化 actor。
- 证据：跨很多控制和游戏任务，强调固定超参和通用性；机器人真机证据不是重点。
- 读完记住：世界模型路线的承诺是样本效率和泛化训练环境，而不是直接替代 VLA。
- 局限：真实机器人中的接触、reset、安全探索和观测噪声更难。

### 10. Open X-Embodiment / RT-X

- 故事：单个实验室的数据不够，单个机器人也不够。Open X 试图把多实验室、多机器人、多任务数据合成一个跨本体学习基座。
- 方法要点：统一多来源机器人轨迹数据，并训练 RT-X 类模型评估跨数据混合的价值。
- 证据：数据集、跨平台任务和模型实验；证据强在“数据混合有价值”，但数据异质性也带来接口复杂度。
- 读完记住：后续 Octo、OpenVLA、SmolVLA 等都站在多机器人数据生态上。
- 局限：数据格式统一不等于动作空间和硬件能力统一；高质量长程数据仍稀缺。

### 11. RT-2

- 故事：RT-2 的关键转折是把机器人动作当成语言 token，让 VLM 像生成文字一样生成动作，从而迁移互联网视觉语言知识。
- 方法要点：把动作离散化为 token，把机器人控制数据和视觉语言数据共同训练。
- 证据：真实机器人评测展示新物体、语义指令和简单推理能力提升。
- 读完记住：RT-2 是 VLA 范式的核心样本：视觉、语言、动作进入同一个序列建模框架。
- 局限：动作 token 化会牺牲连续控制细节；模型闭源，第三方复现和公平比较有限。

### 12. UniSim

- 故事：UniSim 问的是：能否从真实数据学习一个可交互模拟器，用来训练智能体，而不是只在固定仿真器里训练。
- 方法要点：多源数据训练生成式交互世界模型，按高层指令或低层控制生成未来视觉。
- 证据：在视觉语言 planner 和低层策略训练上展示生成经验的作用。
- 读完记住：世界模型短期最实用的价值可能是数据增强和训练环境，而不是完全替代真实世界。
- 局限：视觉真实不等于物理准确；接触丰富任务和闭环误差需要谨慎。

### 13. Mobile ALOHA

- 故事：它把 ALOHA 从桌面双臂扩展到带移动底盘的全身遥操作，任务从短桌面操作走向家务类移动操作。
- 方法要点：低成本 whole-body teleoperation，结合已有 ALOHA 数据和新移动任务数据做 co-training。
- 证据：真实家务任务展示，包括移动、双臂和长程步骤。
- 读完记住：硬件和数据接口的变化会直接决定策略能学哪些任务，移动操作不是简单给桌面策略加轮子。
- 局限：仍高度依赖遥操作数据质量；开放家庭环境的鲁棒性和安全仍远未解决。

### 14. UMI

- 故事：UMI 的聪明之处在于绕开“必须用机器人采集所有示范”的瓶颈。人拿着带传感器的 gripper 在真实场景中演示，再把技能迁移到机器人。
- 方法要点：手持 gripper 收集人类演示，配合相对轨迹动作表示和延迟匹配，让策略更容易部署到不同机器人。
- 证据：真实双臂、动态、长程任务，展示对新环境和物体的泛化。
- 读完记住：数据收集接口本身就是算法创新；便宜、自然、可扩展的数据可能比模型小改更重要。
- 局限：人手、手持设备和机器人之间仍有 embodiment gap；复杂力觉和触觉信息有限。

### 15. DP3

- 故事：2D 图像策略容易在视角、遮挡和空间关系上脆弱。DP3 把 diffusion policy 和简洁 3D 表征结合，强调机器人必须理解三维空间。
- 方法要点：从稀疏点云提取紧凑 3D 表征，再条件化 diffusion action generator。
- 证据：仿真 72 个任务和 4 个真实机器人任务；样本效率和空间泛化较好。
- 读完记住：3D 表征不是可选装饰，它常常决定真实机器人能不能安全泛化。
- 局限：点云质量、相机标定和遮挡仍是工程瓶颈；任务范围仍以操作技能为主。

### 16. ALOHA Unleashed

- 故事：这篇像是在问：如果我们把 ALOHA 数据规模做大，再配上 diffusion policy，能否推进更难的双臂灵巧任务？
- 方法要点：ALOHA 2 平台的大规模遥操作数据，加 Transformer encoder-decoder diffusion policy 和动作分块。
- 证据：5 个真实任务和 3 个仿真任务，包括系鞋带、挂衬衫、齿轮插入等高接触/形变任务。
- 读完记住：复杂操作不一定先靠更华丽的规划，数据规模、合适的遥操作协议和强动作分布模型就能推边界。
- 局限：每个任务仍训练专门策略；规模化数据成本、任务切换和失败恢复还没有完全解决。

### 17. Octo

- 故事：Octo 的定位不是最大的 VLA，而是一个开源、可微调、跨数据的 generalist robot policy 初始化器。
- 方法要点：Transformer diffusion policy，在 Open X-Embodiment 等数据上预训练，支持语言或目标图像条件。
- 证据：多机器人平台和系统消融，强调预训练、架构和微调设计。
- 读完记住：Octo 的价值在“开放初始化器”，让研究者不用从零训练通用策略。
- 局限：新机器人仍常需要微调；零样本泛化不能过度想象。

### 18. OpenVLA

- 故事：OpenVLA 回应了 RT-2 闭源带来的研究瓶颈：如果 VLA 不开放，就很难复现、微调和比较。
- 方法要点：7B VLA，融合视觉编码器和语言模型，在大规模机器人示范上训练动作 token 输出。
- 证据：29 个任务、多机器人本体，报告相对 RT-2-X 的任务成功率优势；开放代码和权重。
- 读完记住：OpenVLA 的里程碑是把 VLA 变成开源研究基线。
- 局限：开源不等于部署容易；算力、动作空间适配、摄像头配置和真实硬件仍是门槛。

### 19. HPT

- 故事：跨本体学习最大的难点是每个机器人的状态、动作、传感器都不同。HPT 试图预训练一个共享 trunk，让不同 embodiment 接到同一个表征空间。
- 方法要点：把异构 proprioception 和视觉输入对齐成 token，再用共享 Transformer 表征支持下游控制。
- 证据：52 个数据集，仿真和真实任务上显示预训练提升。
- 读完记住：跨本体不是“把所有数据丢进一个模型”这么简单，接口对齐本身就是核心问题。
- 局限：共享表征能否覆盖非常不同的硬件和动作约束，还要看更大规模实证。

### 20. RDT-1B

- 故事：RDT 把 diffusion transformer 推到更大参数和双臂基础模型方向，强调用统一动作空间和多源数据训练高容量策略。
- 方法要点：Diffusion Transformer action generator，面向双臂操作和多机器人数据。
- 证据：真实/仿真任务和 benchmark 结果，显示大模型 diffusion policy 的潜力。
- 读完记住：VLA 不是唯一通用策略路线，纯动作生成基础模型也在扩展。
- 局限：双臂操作任务占比高；是否能自然承载开放语言和长程推理，要看与 VLM/VLA 的结合。

### 21. pi0

- 故事：pi0 把 VLA 和 flow matching 结合，目标是既继承 VLM 的语义理解，又生成高频连续动作。
- 方法要点：VLM 作为理解和条件化模块，flow matching action expert 输出动作 chunk。
- 证据：多平台、多任务真实机器人演示和评估，覆盖单臂、双臂、移动操作等。
- 读完记住：pi0 的看点不是名字，而是“VLA + 连续生成式动作头”这条接口设计。
- 局限：数据和训练细节开放度有限；第三方复现和失败边界需要持续观察。

### 22. FAST

- 故事：如果 VLA 要用 next-token prediction 学高频连续动作，朴素分桶 token 会产生又长又相关的 token 序列。FAST 把动作 token 化看成时序压缩问题。
- 方法要点：用离散余弦变换等频域压缩动作序列，再量化和 BPE，得到更短、更可学的动作 token。
- 证据：与 pi0-FAST 结合，报告训练速度显著提升并保持竞争力。
- 读完记住：动作表示是 VLA 的核心瓶颈之一，tokenizer 不是工程细节，而会改变可扩展性。
- 局限：压缩假设可能不适合所有动作分布；高频接触和异常动作的保真度需要细看。

### 23. Cosmos

- 故事：Cosmos 把世界模型从单篇算法推成平台：视频数据处理、tokenizer、预训练世界基础模型和后训练工具。
- 方法要点：World Foundation Model 平台，用于生成、理解和定制物理世界视频模型。
- 证据：技术报告和平台发布，证据更多是系统能力和生态建设，具体机器人策略提升要看下游验证。
- 读完记住：世界模型路线正在平台化，短期价值主要是 synthetic data、仿真和评测。
- 局限：平台宣传色彩较强；“世界模型”是否物理准确，需要任务级证据。

### 24. GR00T N1

- 故事：GR00T N1 把 VLA 放到人形机器人和 NVIDIA Isaac/Cosmos 栈里，强调从仿真、合成数据、微调到部署的完整平台。
- 方法要点：双系统结构，VLM/语言理解模块加 diffusion transformer action head，面向 humanoid 多模态输入和连续动作。
- 证据：论文、官方 repo、仿真和真实 humanoid 任务；平台证据强，独立复现仍要观察。
- 读完记住：GR00T 的重点不是单个模型，而是模型、仿真、数据和硬件部署栈的绑定。
- 局限：生态依赖重；不同 humanoid 硬件的迁移成本和真实安全边界仍不清楚。

### 25. pi0.5

- 故事：pi0.5 继续追问 VLA 如何从专门任务走向开放世界泛化，核心是利用异构机器人数据、语义预测和 web 数据协同训练。
- 方法要点：基于 pi0 的 VLA/flow 架构，增强开放世界任务和多来源训练。
- 证据：真实家庭和多任务设置中的泛化展示；动态较新，需要关注复现和第三方评测。
- 读完记住：VLA 的下一步不是只会更多任务，而是能在未见环境中用语义和物理经验组合行动。
- 局限：开放世界泛化很容易被 demo 选择偏差放大，需要看失败样本和评测协议。

### 26. SmolVLA

- 故事：SmolVLA 的问题意识很清楚：如果 VLA 只能在大公司算力上训练，它就很难形成社区数据飞轮。
- 方法要点：450M 级 VLA，结合轻量 VLM、flow matching decoder、LeRobot 数据和异步推理。
- 证据：仿真和真实 SO-100/SO-101 等低成本平台评测，并开放训练/推理配方。
- 读完记住：小模型路线的意义是降低实践门槛，不是立刻替代最大模型。
- 局限：参数小带来能力边界；复杂长程任务和强语义推理仍需谨慎。

### 27. GigaBrain-0

- 故事：GigaBrain-0 把世界模型生成数据作为 VLA 数据瓶颈的解法：真实机器人数据贵，就用世界模型生成多样训练数据。
- 方法要点：世界模型生成 real2real、人类迁移、视角迁移、sim2real 等数据，训练 VLA，并加入 RGBD 和 embodied CoT 监督。
- 证据：论文报告多个真实任务提升；属于高动态前沿，需追踪代码、数据和复现。
- 读完记住：世界模型短期可能先作为 VLA 数据引擎，而不是直接控制器。
- 局限：生成数据质量、分布偏差和评测公平性是主要风险。

### 28. pi0.6 Model Card

- 故事：model card 不是完整论文，但它记录了 pi0.6 的关键版本事实：骨干、action expert、Knowledge Insulation、推理速度和数据构成。
- 方法要点：SigLIP + Gemma backbone，action expert 输出连续动作，FAST token 和 web/robot 数据协同训练，阻断 action expert 对 VLM 主干的部分梯度影响。
- 证据：版本说明和评测摘要；适合记录事实，不适合替代论文精读。
- 读完记住：对快速迭代模型，model card 是维护知识库版本边界的重要来源。
- 局限：信息密度高但细节有限，需要和 pi0.5、pi*0.6 论文一起读。

### 29. pi*0.6 / RECAP

- 故事：之前许多 VLA 主要靠模仿学习，天花板受示范质量限制。RECAP 的故事是：VLA 可以从部署经验中继续变好。
- 方法要点：先 offline RL 预训练，再用专家介入纠错，最后用自主数据和价值函数做 advantage conditioning，绕开 flow matching 缺少 log-prob 的问题。
- 证据：espresso、laundry、box assembly 等真实任务，报告吞吐量和失败率显著改善。
- 读完记住：这是“VLA + RL 回归”的重要样本，说明部署经验可能成为数据飞轮。
- 局限：人在回路、奖励/价值标注和安全收集成本很高；第三方复现仍需观察。

### 30. GigaBrain-0.5M* / RAMP

- 故事：它把 RECAP 进一步推向世界模型条件 RL：不仅从经验学，还让世界模型帮助策略预判未来。
- 方法要点：在 GigaBrain-0.5 基础上用 RAMP 做 world model-conditioned policy RL。
- 证据：论文报告相对 RECAP 在 laundry、box、espresso 等任务上进一步提升；发布时间近，证据要持续复查。
- 读完记住：2026 年前沿问题正在从“VLA 能不能做”转向“VLA 如何通过世界模型和 RL 越用越好”。
- 局限：论文短、声称强，最需要看公开代码、真实评测协议和独立复现。

### 31. LeRobot

- 故事：LeRobot 不是单个策略模型，而是把数据采集、数据格式、训练、推理、模型分享和低成本硬件连接起来的开源实践栈。
- 方法要点：统一 robot interface、dataset、policy、training/evaluation/deployment pipeline，并连接 Hugging Face Hub。
- 证据：工具链和多硬件实践证据；它的价值是降低复现和实验门槛。
- 读完记住：没有工具链，VLA 和数据飞轮很难进入社区规模；LeRobot 是实践路线的入口。
- 局限：工具链降低门槛，但不能消除硬件标定、安全、延迟、数据质量和部署维护问题。

## 最推荐的阅读顺序

### 如果是初学者

1. SayCan：先理解语言规划为什么要接可执行性。
2. ACT / ALOHA：理解示范数据、低成本硬件和动作分块。
3. Diffusion Policy：理解生成式动作轨迹。
4. RT-2：理解 VLA 的动作 token 思路。
5. OpenVLA 或 SmolVLA：理解开源 VLA 如何实践。
6. LeRobot：进入动手路线。

### 如果要研究策略模型

1. RT-1、RT-2、Open X：读懂数据规模化和 VLA 起源。
2. Octo、OpenVLA、HPT：比较开源通用策略的接口选择。
3. Diffusion Policy、DP3、RDT、pi0、FAST：比较动作头。
4. pi0.5、pi*0.6、GigaBrain：追踪 VLA 后续迭代。

### 如果要判断未来路线

1. DreamerV3、UniSim、Cosmos：理解世界模型的基础承诺。
2. pi*0.6 / RECAP：理解 VLA + RL 为什么重新重要。
3. GigaBrain-0、GigaBrain-0.5M*：观察世界模型作为数据引擎和 RL 引擎的前沿。
4. GR00T：观察大公司平台化路线。

## 关键判断

- 不要把这些论文排成单线替代史。ACT、Diffusion Policy、VLA、World Model 更像系统里的不同接口，常常会组合。
- VLA 的瓶颈不只在模型大小。动作表示、数据质量、跨本体接口、延迟、安全和评测协议同样关键。
- 世界模型短期最现实的角色可能是数据生成、仿真训练和失败预测；直接闭环控制还要解决实时性和物理精度。
- RL 正在回到机器人基础模型叙事中，但它不是“无监督自动变强”。人在回路、奖励定义、安全探索和数据治理都是硬成本。
- 开源生态是 2025-2026 的重要变量。SmolVLA、openpi、GR00T、LeRobot 让研究从“看论文 demo”更接近“可复现、可微调、可部署”。
