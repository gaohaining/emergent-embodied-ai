# Spatial Representation

## 它解决什么

空间表示帮助机器人理解“哪里”和“怎么到达”。没有空间表示，机器人即使识别出杯子，也可能不知道杯子相对夹爪在哪里。

## 常见空间表示

| 表示 | 适合什么 | 局限 |
| --- | --- | --- |
| 2D 图像坐标 | 检测、分割、视觉跟踪 | 不能直接表示真实距离 |
| 深度图 | 单视角距离估计 | 遮挡和传感器噪声明显 |
| 点云 | 3D 几何、抓取、避障 | 数据稀疏，处理成本高 |
| 体素 | 规则 3D 网格 | 分辨率和内存成本冲突 |
| 语义地图 | 导航和任务规划 | 更新和开放世界泛化困难 |
| 对象关系图 | 物体关系和任务推理 | 依赖稳定对象识别 |
| Latent 表征 | 端到端策略学习 | 可解释性和安全约束较弱 |

## 坐标系很重要

机器人系统通常有多个坐标系：

- 相机坐标系。
- 机器人基座坐标系。
- 末端执行器坐标系。
- 世界或地图坐标系。

如果坐标转换错了，模型“看懂了”也可能抓偏。

## 初学者例子

任务：“抓起桌上的杯子。”

系统需要知道：

- 杯子在图像中的位置。
- 杯子在真实 3D 空间中的位置。
- 夹爪当前在哪里。
- 从夹爪到杯子的轨迹是否会碰撞。
- 抓取后杯子是否稳定。
