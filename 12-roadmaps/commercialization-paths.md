# Commercialization Paths

## 判断框架

一个具身智能场景是否容易先落地，通常取决于：

- 任务频率是否高。
- 环境是否可控。
- 失败成本是否低。
- 是否能人工接管。
- 是否有明确付费方。
- 硬件成本能否被收益覆盖。
- 数据能否持续回流。

## Candidate Paths

| 场景 | 为什么可能先落地 | 主要风险 |
| --- | --- | --- |
| 仓储分拣 | 环境相对结构化，任务高频 | 长尾物体和系统集成 |
| 工业辅助操作 | ROI 明确，安全边界可设计 | 柔性任务和换线成本 |
| 商业清洁和巡检 | 任务重复，环境可半结构化 | 异常处理和维护 |
| 远程操作增强 | 可由人兜底，先积累数据 | 人力成本和体验 |
| 实验室自动化 | 环境受控，任务价值高 | 器具多样、可靠性要求高 |
| 家庭服务 | 市场大，任务丰富 | 环境开放、安全和成本极难 |
| 人形通用劳动力 | 想象空间大 | 当前可靠性、成本和安全仍是瓶颈 |

## Likely Sequence

1. 半结构化场景。
2. 人工可接管任务。
3. 高频、低失败成本任务。
4. 逐步增加开放性和自主性。
5. 最后才是复杂家庭和完全开放环境。

## Open Questions

- 哪些场景能最早形成数据飞轮？
- 人形平台会先用于真实劳动，还是先用于数据采集和远程操作？
- 产品价值来自“替代人”，还是先来自“增强人”？
