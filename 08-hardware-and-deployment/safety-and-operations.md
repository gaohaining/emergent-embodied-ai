# 安全与运维

## 它解决什么

部署不是把模型放到机器人上就结束。真实机器人需要安全边界、监控、日志、人工接管和版本管理。

## 基础安全问题

- 机器人是否会碰到人？
- 是否会损坏物体或环境？
- 失败后是否能停止？
- 人是否能随时接管？
- 模型输出是否被低层安全约束限制？

## 运维检查清单

| 项目 | 要检查什么 |
| --- | --- |
| 急停 | 是否有物理和软件急停 |
| 限速 | 是否限制速度、力和工作空间 |
| 日志 | 是否记录观察、动作、模型版本、失败原因 |
| 回放 | 是否能复现实机失败 |
| 接管 | 是否支持人工接管和恢复 |
| 版本 | 是否记录模型、数据和配置版本 |
| 监控 | 是否能发现异常状态 |
| 回滚 | 新模型失败时是否能回到旧版本 |

## 模型部署关注点

- 推理延迟。
- 边缘算力。
- 网络依赖。
- 模型压缩和量化。
- 传感器同步。
- 低层控制频率。
- 故障隔离。

## 记忆点

机器人部署不是“模型准确率”问题，而是系统可靠性问题。任何一个小环节不稳定，都可能让整体能力不可用。

## 开放问题

- 通用机器人模型如何安全在线更新？
- 大模型云端推理和本地控制如何分工？
- 真实部署中的失败数据如何进入训练闭环？
