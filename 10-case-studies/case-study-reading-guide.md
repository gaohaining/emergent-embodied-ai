# 案例研究阅读指南

## 它解决什么

案例研究帮助读者把零散技术环节串起来。看一个系统时，不只看模型名，还要看数据、动作接口、评测和部署条件。

## 阅读顺序

1. 它做什么任务？
2. 用什么机器人平台？
3. 输入是什么？
4. 输出是什么？
5. 数据从哪里来？
6. 模型或系统结构是什么？
7. 如何评测？
8. 有哪些失败和限制？
9. 它改变了哪个技术判断？

## 技术环节映射

| 环节 | 需要记录 |
| --- | --- |
| 感知 | 用了哪些传感器和视觉模型 |
| 表征 | 如何表示状态、空间、动作 |
| 数据 | 数据来源、规模、质量、许可 |
| 策略 | 学习方法、模型输入输出 |
| 规划控制 | 是否分层、是否有技能库 |
| 仿真评测 | benchmark、实机任务、成功率 |
| 部署 | 延迟、安全、人工接管、成本 |

## 输出模板

读完一个案例后，用 5 句话总结：

1. 这个系统想解决什么任务。
2. 它最重要的技术贡献是什么。
3. 它依赖什么数据和硬件。
4. 它的证据强在哪里，弱在哪里。
5. 我从中学到的系统设计原则是什么。
