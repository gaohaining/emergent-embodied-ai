# Task Types

## 它解决什么

具身智能不是一个单一任务，而是一组不同难度、不同约束的任务。初学者先分清任务类型，读论文和看 demo 时才不会把“会抓一个方块”误解成“能做家务”。

## 常见任务类型

| 任务类型 | 例子 | 主要难点 | 常见平台 |
| --- | --- | --- | --- |
| 抓取 | 抓杯子、抓方块、分拣物体 | 物体姿态、遮挡、抓取稳定性 | 机械臂、夹爪 |
| 放置 | 把物体放到目标区域 | 空间关系、碰撞、放置后稳定性 | 机械臂 |
| 操作 | 开抽屉、拧瓶盖、插拔物体 | 接触丰富、力控、长程步骤 | 机械臂、灵巧手 |
| 导航 | 到达目标位置、避障 | 地图、定位、动态障碍 | 移动机器人、四足 |
| 移动操作 | 边移动边操作物体 | 导航和操作耦合 | 移动机械臂、人形 |
| 人机协作 | 递物、跟随、共同搬运 | 意图理解、安全距离、实时反馈 | 服务机器人、人形 |
| 长程任务 | 做早餐、整理桌面 | 子任务分解、失败恢复、状态记忆 | 组合系统 |

## 难度从哪里来

- 物理接触：接触越复杂，越难只靠视觉和语言解决。
- 时间长度：任务越长，错误越容易累积。
- 环境开放性：场景、物体和指令越开放，泛化越难。
- 安全约束：靠近人、贵重物体或不确定环境时，容错空间更小。
- 数据来源：任务越复杂，越难获得高质量示范数据。

## 初学者读材料时先问

- 任务是在仿真里，还是真实机器人上？
- 环境是固定的，还是有新物体、新布局、新指令？
- 系统是完成一个技能，还是完成多步任务？
- 有没有失败案例和统计评测？
- 人类是否在过程中干预？

## Open Questions

- 哪些任务最适合作为通用具身智能的早期 benchmark？
- 长程任务应该按最终成功率评测，还是拆解成感知、规划、控制和恢复能力？
- 人形机器人是否会改变任务类型的优先级？
