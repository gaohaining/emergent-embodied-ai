# 系统闭环

## 具身智能的闭环

具身智能和普通多模态 AI 最大的区别在于：它的输出会改变世界。

```text
观察 → 理解 → 决策 → 动作 → 环境变化 → 新观察
```

如果动作失败，系统需要发现失败并恢复；如果环境变化，系统需要重新感知和规划；如果长期部署，系统还需要从日志和反馈中学习。

## 每一环在做什么

| 环节 | 作用 | 例子 |
| --- | --- | --- |
| 观察 | 获取环境信息 | 相机、深度相机、触觉、本体状态 |
| 理解 | 判断对象、位置、关系和任务 | 杯子在桌上，水槽在右侧 |
| 决策 | 选择下一步做什么 | 先移动到杯子旁边 |
| 动作 | 生成机器人可执行控制 | 机械臂轨迹、夹爪开合 |
| 反馈 | 判断动作结果 | 是否抓住杯子，是否碰撞 |
| 学习 | 用数据改进策略 | 从成功和失败 episode 中更新模型 |

## 为什么这件事难

- 真实世界是连续、嘈杂、会变化的。
- 机器人动作有延迟、误差和安全约束。
- 物体接触、遮挡、滑动和变形很难建模。
- 长程任务中一个小错误会影响后续所有步骤。

## 读论文时怎么定位

看到一篇论文时，先问：

- 它主要改进闭环里的哪一环？
- 它是在仿真里验证，还是在真实机器人上验证？
- 它解决的是单个技能，还是长程任务？
- 它的失败模式是什么？
