# Language Policy

本仓库支持中英文，但不采用“一开始所有页面双份维护”的方式。原因很简单：内容还在快速演化，双份全文很容易不同步。

## 默认语言

- 中文是主维护语言。
- 英文用于关键入口、术语、缩写、论文名、项目名和重要页面译文。
- 面向初学者的解释优先写清楚，而不是追求逐字翻译。

## 页面命名

推荐使用相邻译文文件：

```text
README.md        # 中文主入口
README.en.md     # 英文入口
topic.md         # 中文主页面
topic.en.md      # 英文译文，可选
```

暂不建议建立完整的 `en/` 镜像目录，除非英文内容已经稳定到需要独立维护。

## 页面内容规则

中文页面：

- 标题可以用英文技术词，但正文用中文解释。
- 第一次出现重要术语时，写成 `中文术语（English Term, Acronym）`。
- 保留论文、模型、数据集、benchmark 和项目的英文原名。
- 初学者解释优先，避免堆缩写。

英文页面：

- 不要求逐句翻译，可以是面向英文读者的等价说明。
- 链接回中文主页面或相关技术目录。
- 如果英文页内容落后，顶部标注 `Status: draft` 或 `Status: partial translation`。

## 术语表

[../11-glossary/README.md](../11-glossary/README.md) 是中英文术语对齐的主位置。新增重要概念时，优先补：

- 中文术语。
- 英文术语。
- 缩写。
- 初学者工作定义。
- 相关技术环节页面。

## 翻译优先级

优先翻译这些页面：

1. 根目录 README。
2. 新手入口和系统总览。
3. 术语表。
4. 每个技术环节的 README。
5. 代表性案例。

不急着翻译：

- 临时 inbox。
- 周度回顾。
- 尚未沉淀的个人笔记。
- 过期风险高的动态材料。

## 维护检查

新增或修改英文内容时，检查：

- 英文页是否链接回中文主入口。
- 术语是否和 glossary 一致。
- 页面是否说明自己是完整译文还是摘要。
- 重要中文页面是否仍然是事实来源。
